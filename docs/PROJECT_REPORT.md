> تاریخ: خرداد ۱۴۰۵ | نسخه فعلی: Phase 2

## ۱. معرفی پروژه

**TelBattle** یک بازی کارت جمع‌آوری (Collectible Card Game) داخل تلگرام است که کاربران می‌توانند کارت‌های شخصیت‌های معروف سینما، انیمه، بازی‌های ویدیویی و فرهنگ ایرانی را جمع‌آوری کرده و با یکدیگر مبارزه کنند.

**هدف اصلی:** ساخت یک بازی کارت گرافیکی و تعاملی داخل تلگرام که هم از طریق گروه‌ها و هم به صورت انفرادی قابل بازی باشد.

## ۴. سیستم PvP (مبارزه ۳ راوندی)

### فلوی کامل مبارزه
```
۱. چلنجر در گروه /fight می‌زند
۲. پیام چالش در گروه ارسال می‌شود
۳. حریف یکی از دو گزینه انتخاب می‌کند:
   - ✊ قبول (نرمال) → هر دو در پیوی کارت انتخاب می‌کنند
   - 🎲 قبول (تصادفی) → کارت‌ها تصادفی انتخاب می‌شوند
۴. انتخاب زمین بازی (Arena):
   - اگر rarity کارت‌ها برابر → زمین تصادفی
   - اگر یکی ضعیف‌تر → بازیکن ضعیف‌تر زمین را انتخاب می‌کند
۵. ۳ راوند: هر راوند یک آمار انتخاب می‌شود (Stat Locking — آمار تکراری مجاز نیست)
۶. در هر راوند: آمار انتخابی + Boost زمین محاسبه می‌شود
۷. برنده ۲ راوند از ۳ پیروز است
```

### زمین‌های بازی (Arena)
| زمین | Boost |
|------|-------|
| ⚡ عرصه قدرت | کارت‌های POWER_TYPE +1 |
| 🏃 پیست سرعت | کارت‌های SPEED_TYPE +1 |
| 🧠 اتاق فکر | کارت‌های IQ_TYPE +1 |
| ⭐ صحنه | کارت‌های POPULARITY_TYPE +1 |

### پاداش مبارزه
- برنده: +10 تا +20 امتیاز + XP + Tier Points
- بازنده: -1 قلب + XP (کمتر) - Tier Points

### Card Missions
- هر کارت Epic یک ماموریت خاص دارد
- تکمیل ماموریت → تبدیل خودکار به Legend

### Skins
- اسکین‌های کارت با سکه قابل خرید
- نمایش استیکر متفاوت در مبارزه

## ۸. مرحله بعدی: Mini App تلگرام

### چرا Mini App؟
کاربران می‌توانند بدون گروه، مستقیم از پیوی ربات بازی کنند. رابط گرافیکی کامل.

### Mini App چیست؟
یک وب‌اپ (HTML/CSS/JavaScript) که با کلیک روی دکمه داخل تلگرام باز می‌شود. کاملاً داخل تلگرام. نیاز به نصب ندارد.

### معماری پیشنهادی

```
┌─────────────────────────────────────────┐
│           Telegram Client                │
│  ┌─────────────────────────────────┐   │
│  │     Mini App (WebApp)            │   │
│  │  React.js / Vue.js               │   │
│  │  - نمایش کارت‌ها با تصویر       │   │
│  │  - PvP گرافیکی                  │   │
│  │  - Solo vs AI                   │   │
│  │  - پروفایل و لیدربرد           │   │
│  └────────────┬────────────────────┘   │
└───────────────┼─────────────────────────┘
                │ HTTP/WebSocket
                ▼
┌─────────────────────────────────────────┐
│         Python Backend                   │
│  - همان game_core.py                    │
│  - Flask REST API (موجود)              │
│  - WebSocket برای PvP real-time         │
└─────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────┐
│         SQLite Database                  │
│         (game_bot.db)                    │
└─────────────────────────────────────────┘
```

### مراحل توسعه Mini App

**مرحله ۱ (۱-۲ هفته):** Mini App ساده
- نمایش کارت‌ها با تصویر استیکر
- پروفایل کاربر
- لیدربرد
- اتصال به Flask API فعلی

**مرحله ۲ (۲-۴ هفته):** Solo PvP
- مبارزه با AI (رایانه)
- انیمیشن کارت flip
- نمایش زمین بازی

**مرحله ۳ (۴-۸ هفته):** Multiplayer گرافیکی
- PvP real-time از طریق WebSocket
- آنلاین matchmaking
- جوایز و پاداش

### نیازمندی‌های فنی
| آیتم | گزینه رایگان | گزینه پولی |
|------|-------------|-----------|
| Frontend Hosting | Vercel / Netlify | - |
| Backend Hosting | Railway / Render | VPS |
| Database | SQLite (فعلی) | PostgreSQL |
| WebSocket | Python asyncio | - |

### Stack پیشنهادی برای Mini App
- **Frontend:** React.js یا Vue.js + Tailwind CSS
- **Telegram SDK:** @tma.js/sdk یا @telegram-apps/sdk
- **Backend:** همان Python/Flask با اضافه کردن WebSocket
- **State Management:** Zustand / Pinia

## ۹. سوالات برای مشاوره با AI های دیگر

### سوال ۱ — معماری
> "می‌خواهیم یک بازی کارت (Collectible Card Game) که الان به صورت text-based در تلگرام اجرا می‌شود را به یک Mini App گرافیکی تبدیل کنیم. Backend Python/Flask/SQLite است. بهترین معماری و stack frontend چیست؟"

### سوال ۲ — Solo AI
> "در یک بازی کارت ۳ راوندی (هر کارت ۴ آمار دارد: power/speed/iq/popularity)، چطور یک AI ساده طراحی کنیم که چالش‌برانگیز باشد اما به منابع زیادی نیاز نداشته باشد؟"

### سوال ۳ — Real-time PvP
> "برای PvP real-time در یک Mini App تلگرام، بهترین روش برای sync state بین دو بازیکن چیست؟ WebSocket، polling یا long-polling؟"

### سوال ۴ — Refactor
> "یک فایل Python 4700 خطی (telegram bot) داریم که شامل 79 handler است و همه به یک کلاس اصلی متصل هستند. بهترین روش refactor با Mixin pattern چیست بدون اینکه functionality شکسته شود؟"
