# 🌐 راهنمای راه‌اندازی پنل مدیریت کارت‌ها

## 🔍 مشکل فعلی
پنل مدیریت کارت‌ها وصل نمیشه چون احتمالاً:
1. سرویس `web_api.py` روی سرور اجرا نشده
2. پورت 5000 بسته است یا فایروال مسدودش کرده
3. فایل‌های مورد نیاز روی سرور نیستند

## ✅ راه‌حل‌ها

### گزینه 1: بررسی وضعیت فعلی (توصیه می‌شود)
```batch
check_web_panel_status.bat
```

این اسکریپت نشون میده:
- ✅ آیا پنل وب در حال اجرا است؟
- ✅ پورت 5000 باز است؟
- ✅ چه فایل‌هایی روی سرور هستند؟
- ✅ آخرین لاگ‌های بات

### گزینه 2: راه‌اندازی پنل وب
```batch
start_web_panel_remote.bat
```

این اسکریپت:
1. فایل `web_api.py` را به سرور آپلود می‌کند
2. اسکریپت راه‌اندازی را اجرا می‌کند
3. پنل را در پس‌زمینه اجرا می‌کند

### گزینه 3: راه‌اندازی دستی (اگر اسکریپت‌ها کار نکردند)

## 🔧 عیب‌یابی

### مشکل 1: پورت 5000 در دسترس نیست
**علت:** فایروال سرور پورت را مسدود کرده

**راه‌حل:**
```bash
# اتصال به سرور
ssh root@195.248.243.122

# باز کردن پورت 5000
ufw allow 5000/tcp

# یا اگر از iptables استفاده می‌کنی:
iptables -A INPUT -p tcp --dport 5000 -j ACCEPT
```

### مشکل 2: خطای "Address already in use"
**علت:** پورت 5000 قبلاً استفاده شده

**راه‌حل:**
```bash
# پیدا کردن پروسه
lsof -i :5000

# یا
netstat -tulpn | grep 5000

# متوقف کردن پروسه
kill -9 <PID>
```

### مشکل 3: خطای "ModuleNotFoundError"
**علت:** کتابخانه‌های پایتون نصب نیستند

**راه‌حل:**
```bash
cd "/root/card game"
pip3 install flask flask-cors
```

### مشکل 4: بات تلگرام آسیب می‌بینه
**نگران نباش!** پنل وب و بات تلگرام مستقل از هم هستند:
- بات تلگرام: `telegram_bot.py`
- پنل وب: `web_api.py`

هر دو می‌تونن همزمان اجرا بشن بدون مشکل.

## 🔄 راه‌اندازی مجدد همه سرویس‌ها

اگر می‌خوای همه چیز رو از نو راه‌اندازی کنی:

```bash

pkill -9 -f web_api.py

# راه‌اندازی بات تلگرام
cd "/root/card game"
nohup python3 telegram_bot.py > bot.log 2>&1 &

# راه‌اندازی پنل وب
nohup python3 web_api.py > web_panel.log 2>&1 &

# بررسی وضعیت
ps aux | grep python
```
